# AWS S3 Setup Guide

Your project is now configured to use AWS S3 for file storage and PDF uploads. Follow these steps to enable it:

## Step 1: Get Your AWS Credentials

1. Go to [AWS IAM Console](https://console.aws.amazon.com/iamv2)
2. Click **Users** in the left sidebar
3. Create a new user or use an existing one
4. Create an **Access Key** for programmatic access:
   - Click the user → **Security credentials** tab
   - Scroll to **Access keys** → **Create access key**
   - Choose **Application running outside AWS** → **Create access key**
   - Copy the **Access Key ID** and **Secret Access Key**

## Step 2: Attach S3 Permissions

1. In the same IAM user, go to **Add permissions** → **Attach policies directly**
2. Search for and select: **AmazonS3FullAccess**
3. Click **Add permissions**

## Step 3: Update Your .env File

Open `/backend/.env` and replace:

```env
USE_S3=true
AWS_ACCESS_KEY_ID=your_actual_access_key_id_here
AWS_SECRET_ACCESS_KEY=your_actual_secret_access_key_here
AWS_REGION=us-east-1
AWS_S3_BUCKET_NAME=assignment-creator
```

Get the values from Step 1 above.

## Step 4: Test the Connection

Run your backend:

```bash
cd backend
npm run dev:all
```

You should see:
```
✅ AWS S3 Connection verified. Bucket: assignment-creator
```

## How It Works Now

### File Upload
- User uploads a file via the frontend
- File is automatically uploaded to S3 at: `s3://assignment-creator/uploads/assignments/filename.pdf`
- Database stores the S3 URL (not local path)

### PDF Generation
- Questions are generated by Gemini AI
- PDF is created in-memory and uploaded to S3
- Database stores the S3 URL

### File Download
- User clicks download
- Browser redirects to the S3 URL
- S3 serves the file directly (no need to download via your backend)

## Deployment

When deploying to production (Vercel, Railway, Render, etc.):

1. Set environment variables in your deployment platform:
   - `USE_S3=true`
   - `AWS_ACCESS_KEY_ID=xxx`
   - `AWS_SECRET_ACCESS_KEY=xxx`
   - `AWS_S3_BUCKET_NAME=assignment-creator`

2. Your application will automatically:
   - Store all files in S3 (no ephemeral storage issues)
   - Scale horizontally (files accessible from any server)
   - Reduce backend load (S3 handles file serving)

## S3 Bucket Permissions

Your bucket should have the following CORS configuration (if serving files from browser):

```json
[
  {
    "AllowedHeaders": ["*"],
    "AllowedMethods": ["GET", "PUT", "POST"],
    "AllowedOrigins": ["*"],
    "ExposeHeaders": ["ETag"],
    "MaxAgeSeconds": 3000
  }
]
```

Go to S3 Console → Your Bucket → **Permissions** → **CORS** to set this.

## Cost Estimation

For typical usage:
- **Storage**: ~$0.023 per GB/month (very cheap for PDFs)
- **Data Transfer**: ~$0.09 per GB (only pay if you download files)
- **Requests**: ~$0.0004 per 1000 uploads

See [AWS S3 Pricing](https://aws.amazon.com/s3/pricing/) for details.

## Troubleshooting

### "InvalidBucketName" Error
- Ensure bucket name is lowercase with no spaces
- Verify bucket exists in your AWS console

### "Access Denied" Error
- Check your AWS credentials are correct
- Verify the IAM user has `AmazonS3FullAccess` policy
- Check that bucket is in the correct region

### Files Not Uploading
- Check CloudWatch logs in AWS Console
- Ensure `USE_S3=true` in .env
- Verify bucket name matches `AWS_S3_BUCKET_NAME`

## Development (Local Testing with Local Storage)

To use local file storage for development:

```env
USE_S3=false
```

Files will be saved to the `uploads/` folder and served via Express.static().

---

**Note**: Never commit your `AWS_SECRET_ACCESS_KEY` to Git. Always use environment variables!
